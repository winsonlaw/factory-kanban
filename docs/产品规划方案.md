# 工厂生产看板系统 — 完整规划方案

## Context

集团级多工厂（10+厂区）生产可视化系统，全新搭建从PLC数据采集到车间大屏展示的完整链路。
每个车间电视（小主机 HDMI 接屏）运行 Electron 应用，分两层展示：
- **车间屏**：总览该车间所有产线 OEE、状态、产量 vs 目标
- **产线屏**：逐站过机数据、效率分析、实时动态流转动画

---

## 系统分层架构

```
设备/PLC 层
  ↓  MQTT / OPC-UA / Modbus TCP / RS485
边缘网关（每车间一台工控机/树莓派）
  ↓  MQTT over TLS
中心数据平台（云/本地机房）
  ├── EMQX Broker（集群，支持10万+连接）
  ├── TDengine（时序数据库，亿级采集点）
  ├── PostgreSQL（工厂/产线/站位基础数据）
  ├── Redis（实时状态缓存）
  └── Node.js API Server（REST + WebSocket）
        ↓  WebSocket 实时推送
Electron 大屏应用（每台展示主机）
  ├── 车间总览视图 (Level 1)
  └── 产线详情视图 (Level 2)  ← 多产线时自动轮播
```

## 模块一：数据采集层

### 边缘网关（Edge Gateway）
- **硬件**：工控机 / 树莓派 4 / x86 工控板（每车间一台）
- **软件**：Node-RED 或自研 Go 采集服务
- **支持协议**：
  - Modbus TCP/RTU（最常见 PLC）
  - OPC-UA（西门子、B&R）
  - MQTT（已支持的智能设备）
  - Serial RS485（老旧设备）
- **本地缓冲**：网络断开时本地存储 30 分钟数据，恢复后补传

### 采集数据字段（每站每次推送）
```json
{
  "factory_id": "F01",
  "zone_id": "Z03",
  "line_id": "L02",
  "station_id": "S05",
  "timestamp": 1717300000000,
  "pass_count": 1,        // 过机数（良品）
  "fail_count": 0,        // 不良数
  "cycle_time_ms": 4800,  // 本次节拍时间
  "equipment_status": 1,  // 0=停机 1=运行 2=待机 3=报警
  "custom_fields": {}     // 扩展字段
}
```

## 模块二：中心数据平台

### 技术选型
| 组件 | 选型 | 理由 |
|------|------|------|
| MQTT Broker | EMQX Enterprise | 集群高可用，10万+并发，内置规则引擎 |
| 时序数据库 | TDengine | 超高写入性能，按设备自动分片，SQL 接口 |
| 业务数据库 | PostgreSQL | 工厂/产线/班次/目标等结构化数据 |
| 缓存 | Redis | 最新状态快照，秒级刷新，WebSocket 扇出 |
| API 服务 | Node.js + Fastify | WebSocket 实时推送，REST 历史查询 |
| 消息队列 | Kafka（大型可选） | 数据削峰，跨系统集成 |

### 核心 API 接口
- `WS /ws/zone/:zoneId` — 车间实时数据流
- `WS /ws/line/:lineId` — 产线实时数据流
- `GET /api/shift/summary` — 班次统计
- `GET /api/station/history` — 站位历史过机
- `GET /api/oee/:lineId` — OEE 计算结果

### OEE 计算逻辑
```
OEE = 可用率(A) × 表现率(P) × 质量率(Q)
A = 实际运行时间 / 计划运行时间
P = (实际产量 × 理论节拍) / 实际运行时间
Q = 良品数 / 总产出数
```

## 模块三：Electron 大屏应用

### 技术栈
| 层 | 技术 |
|----|------|
| 框架 | Electron 28 + React 18 + TypeScript |
| 样式 | TailwindCSS 3 + CSS Variables |
| 图表 | Apache ECharts（工业大屏最强） |
| 动画 | Framer Motion + CSS Animation |
| 状态 | Zustand + WebSocket 实时同步 |
| 通信 | ws（WebSocket 客户端） |
| 构建 | Vite + electron-builder |

### 启动行为
- 开机自启（Electron autolaunch）
- 自动全屏（`mainWindow.setFullScreen(true)`）
- 禁止右键菜单、禁止 DevTools（生产模式）
- 网络断开时显示"数据连接中…"遮罩，自动重连

### 产线流动画效果
- 每个站位是一个卡片，站间用动态粒子流连接（CSS animation / Canvas）
- 粒子流速度 = 当前节拍的反比（节拍快 → 粒子快）
- 瓶颈站（节拍最慢）高亮边框闪烁警示
- 设备停机时该站粒子停止流动，前序站积压粒子堆积动画

### 自动轮播逻辑
```
若车间有多条产线：
  产线屏 A → 30秒 → 产线屏 B → 30秒 → ... → 车间总览 60秒 → 循环
```
支持管理端远程控制定屏/切换。

## 部署方案

### 中心平台（云服务器 / 本地机房）
```yaml
# docker-compose.yml
services:
  emqx:        # MQTT Broker
  tdengine:    # 时序数据库
  postgres:    # 业务数据库
  redis:       # 状态缓存
  api-server:  # Node.js 后端
  admin-web:   # Nginx 托管管理后台
```

### 大屏主机（车间侧）
- Windows: `electron-builder` 打包 `.exe` 安装包，配置开机自启
- Android: `capacitor` 封装 Web 或原生 APK（次选）
- 配置文件：`config.json`（指定所属车间 ID + 后端地址）
- 更新：后端推送新版本通知，Electron 自动下载安装

### 网络架构
- 车间内局域网 → 工业交换机 → 边缘网关
- 边缘网关 → 专线/VPN → 中心平台
- 中心平台 → 局域网/Internet → 大屏主机（WebSocket）

## 验收标准

- 数据延迟：设备产生数据 → 大屏刷新 ≤ 3 秒
- 并发：支持 100 台大屏同时在线，单台主机 CPU < 20%
- 可靠性：边缘断网后恢复，数据零丢失补传
- 视觉：1080p 大屏，字体/图表清晰可读，3 米外辨识核心数据
- OEE 精度：与人工统计误差 < 0.5%
