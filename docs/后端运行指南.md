# 后端运行指南

> 配套代码：`packages/data-platform`（中心平台）、`packages/edge-gateway`（边缘网关）。
> 系统默认**免外部基础设施**即可仿真运行：内置 MQTT broker、mock MES、内存存储。

---

## 一、架构回顾

```
edge-gateway ──MQTT(canonical)──▶ EMQX ──▶ data-platform ──WS──▶ 大屏(display-app)
 (设备采集/仿真)                          (OEE/Pacing/MES对接)
                                            ▲
                                  REST 拉工单 / Webhook 回传
                                            │
                                           MES
```

| 包 | 职责 | 技术 |
|----|------|------|
| `edge-gateway` | 多协议采集 → 规范化 → 离线补传 → MQTT 上报 | Node + mqtt（+ 可选 modbus-serial） |
| `data-platform` | MQTT 接入 → OEE/Pacing 计算 → MES 对接 → WS 推送 | Node + Fastify + zod + ioredis |

---

## 二、本地仿真运行（零基础设施，3 个终端）

```bash
# ① 启动内置 MQTT broker（aedes，仅本地开发用）
cd packages/data-platform && npm install
npm run broker                       # mqtt://localhost:1883

# ② 启动中心数据平台
npm start                            # http://localhost:8080 ; ws .../ws/workshop/W01

# ③ 启动仿真边缘网关
cd packages/edge-gateway && npm install
SIM_SPEED=0.05 npm start             # 34 个站位仿真上报
```

验证：

```bash
curl -s --noproxy '*' http://127.0.0.1:8080/api/workshop/W01 | head -c 300
# 或浏览器看 WS：ws://localhost:8080/ws/workshop/W01
```

> `SIM_SPEED` 为节拍缩放，0.05 = 加速约 20 倍便于观察；设 1 为真实节拍。

### 让大屏接入实时后端

```bash
cd packages/display-app
VITE_WS_URL=ws://localhost:8080/ws/workshop/W01 npm run dev
```

未设 `VITE_WS_URL` 时大屏回退内置 mock 演示数据。

## 三、Docker 一键部署（含真实 EMQX/Redis/PostgreSQL/TDengine）

```bash
cd docker
docker compose up -d
# EMQX 1883/18083 · data-platform 8080 · PostgreSQL 5432 · Redis 6379 · TDengine 6041
docker compose logs -f data-platform
```

compose 默认带一个仿真 `edge-gateway`，启动即有数据。接入真实设备时：删除该服务，由现场网关连 `emqx:1883`。

## 四、关键环境变量

### data-platform

| 变量 | 默认 | 说明 |
|------|------|------|
| `STORAGE` | `memory` | `real` 启用 Redis 快照 |
| `HTTP_PORT` | `8080` | HTTP/WS 端口 |
| `MQTT_ENABLED` / `MQTT_URL` | `true` / `mqtt://localhost:1883` | EMQX 接入 |
| `MES_MODE` | `mock` | `rest` 接真实 MES |
| `MES_BASE_URL` / `MES_API_KEY` | — | 真实 MES 地址与鉴权 |
| `MES_CALLBACK_URL` | — | 生产事件回传地址 |
| `MES_WEBHOOK_SECRET` | `dev-secret` | 入站 Webhook HMAC 验签 |
| `SNAPSHOT_INTERVAL_MS` | `1000` | 快照计算/推送节流 |
| `TAKT_WARN_RATIO` | `1.15` | 节拍超标报警阈值 |

### edge-gateway

| 变量 | 默认 | 说明 |
|------|------|------|
| `GATEWAY_MODE` | `simulator` | `modbus` 走真实 PLC |
| `MQTT_URL` | `mqtt://localhost:1883` | 上报地址 |
| `FACTORY_ID` / `ZONE_ID` / `GATEWAY_ID` | `F01`/`Z01`/`GW-Z01-01` | 网关身份 |
| `SIM_SPEED` | `0.05` | 仿真节拍缩放 |
| `SIM_FAIL_RATE` / `SIM_ALARM_CHANCE` | `0.01` / `0.002` | 仿真不良率/报警概率 |
| `BUFFER_FILE` | `/tmp/fk-gateway-buffer.jsonl` | 离线补传缓冲 |

## 五、HTTP / WS 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |
| GET | `/api/workshop/:id` | 取最新车间快照（REST 兜底） |
| WS | `/ws/workshop/:id` | 实时快照推送（连接即推一帧） |
| POST | `/mes/webhook/orders` | MES 工单变更入站（HMAC 验签） |

WS 推送帧格式：`{ "type": "workshop", "data": WorkshopData }`，结构与 display-app 一致，可直接渲染。

---

## 六、接真实设备（Modbus）

1. `cd packages/edge-gateway && npm i modbus-serial`
2. 在 `src/drivers/modbus.ts` 的 `ModbusProfile[]` 配置各设备连接与寄存器映射
3. `GATEWAY_MODE=modbus npm start`

其他协议（OPC-UA / SECS-GEM / RS485）按同样的 `Driver` 接口扩展，参见《对外接口与通道设计》南向部分。
