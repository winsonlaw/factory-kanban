# 生产看板信息架构设计文档（定稿）

> 版本 v1.0 · 面向四类使用者：车间主任 / 现场调度 / 产线组长 / 对外参观
> 本文档是后续所有看板开发的蓝图。先对齐"展示什么、为谁展示、如何判断"，再写代码。

---

## 第一部分：设计哲学

### 1.1 核心原则 —— 从"展示数据"升级到"支撑决策"

> ❌ 反面：满屏状态值（OEE 87%、产量 1247、良率 98.2%），看的人要自己心算"这是好还是坏"。
> ✅ 正面：每个数字旁边给出**判断**——红黄绿、欠产/超产、落后多少分钟、已持续多久。

一块好的生产看板，使用者扫一眼必须能回答三个问题（源自丰田 Andon 安灯思维）：

| 问题 | 对应信息 | 视觉手段 |
|------|---------|---------|
| **① 现在正常吗？** | 设备/产线状态、异常报警 | 红黄绿灯、闪烁 |
| **② 我们落后了吗？** | Pacing：应产 vs 实产、欠产件数、落后分钟 | 大数字、进度差 |
| **③ 问题卡在哪？** | 瓶颈站、停机/缺陷 Pareto | 高亮、排行榜 |

### 1.2 信息三层金字塔

每个视图的信息都要分三层，决定字号和位置：

```
        ┌─────────────────┐
        │  一眼层 (3米外)  │  ← 1-2个核心判断，超大字+红黄绿。例：达成灯、欠产数
        ├─────────────────┤
        │  三秒层 (1米内)  │  ← 5-8个关键指标，中字号。例：OEE、产量、节拍
        ├─────────────────┤
        │  深看层 (走近看) │  ← 明细、趋势、Pareto。小字号、图表
        └─────────────────┘
```

### 1.3 颜色语义（全系统统一）

| 颜色 | 含义 | 触发逻辑 |
|------|------|---------|
| 🟢 绿 | 正常/达标/超前 | OEE≥85%、达成率≥100%、节拍≤Takt |
| 🟡 黄 | 预警/临界 | OEE 60-85%、落后≤10%、节拍超标 5-15% |
| 🔴 红 | 异常/未达/落后 | OEE<60%、停机、节拍超标>15%、连续不良 |
| ⚪ 灰 | 停机/无数据/未排产 | 设备离线、计划外 |

## 第二部分：指标词典（数据基础）

> 所有视图的指标都从这里取。每个指标含：定义、公式、数据来源、目标/阈值。
> 「来源」标注后端如何得到：⚙️PLC直采 / 🧮系统计算 / 📋基础数据配置

### 2.1 产量与节奏类（最核心）

| 指标 | 公式 / 定义 | 单位 | 来源 | 目标/阈值 |
|------|-----------|------|------|----------|
| **本班产量** | 当班累计良品过机数 | 件 | ⚙️ | — |
| **计划产量** | 当班排产目标 | 件 | 📋 | — |
| **应产数 (Pacing基准)** | (当前时间-班次开始)/Takt | 件 | 🧮 | — |
| **欠产/超产** | 实产 - 应产 | 件 | 🧮 | ≥0 为绿 |
| **落后/领先时间** | 欠产 × Takt | 分钟 | 🧮 | 落后>15min 红 |
| **节拍 CT (Cycle Time)** | 单件实际加工时间 | 秒 | ⚙️ | — |
| **产距时间 Takt** | 当班可用时间 / 计划产量 | 秒 | 🧮 | CT≤Takt 才达标 |
| **节拍达成率** | Takt / CT × 100% | % | 🧮 | ≥100% 绿 |
| **每小时产出 UPH** | 每小时良品数 | 件/h | 🧮 | vs 标准UPH |

> ⭐ **Pacing 是整个看板最该放 C 位的概念**：它直接回答"我们落后了吗"。
> 例："应产 1400，实产 1247，**欠产 153 件 · 落后 38 分钟**"

### 2.2 效率类（OEE 体系）

| 指标 | 公式 | 来源 | 目标 |
|------|------|------|------|
| **OEE** | 可用率 × 表现率 × 质量率 | 🧮 | ≥85%(世界级) |
| **可用率 A** | 实际运行时间 / 计划运行时间 | 🧮 | ≥90% |
| **表现率 P** | (实产 × 理论节拍) / 实际运行时间 | 🧮 | ≥95% |
| **质量率 Q** | 良品数 / 总产出 | 🧮 | ≥99% |
| **TEEP** | OEE × 时间利用率(含未排产) | 🧮 | 设备总潜力 |
| **OEE 六大损失** | 停机/换型/小停顿/速度/不良/启动 损失 | 🧮 | 改善靶点 |

> 💡 OEE **瞬时值意义不大，趋势和损失分解才有用**。看板要给7天趋势线+目标线。

### 2.3 质量类

| 指标 | 公式 / 定义 | 来源 | 说明 |
|------|-----------|------|------|
| **良率** | 良品 / 总产出 | 🧮 | 基础 |
| **直通率 FPY** | 一次通过(免返工)数 / 投入数 | 🧮 | 比良率更真实 |
| **滚动直通率 RTY** | 各站 FPY 连乘 | 🧮 | 全线真实质量 |
| **不良数 / 不良率** | 当班不良件数 / 占比 | ⚙️🧮 | — |
| **缺陷 Pareto** | 按缺陷类型(开短路/偏移/缺件)排行 | 🧮 | ⭐告诉组长改什么 |
| **连续不良报警** | 同一缺陷连续≥N件 | 🧮 | ⭐工艺异常,弹窗级 |

> 💡 不良分布要按**缺陷原因**分类，不是按站位分。"开短路 12 件"比"贴片机不良 12 件"有用得多。

### 2.4 设备可靠性类

| 指标 | 公式 | 来源 | 说明 |
|------|------|------|------|
| **MTBF** 平均故障间隔 | 运行时间 / 故障次数 | 🧮 | 越大越好 |
| **MTTR** 平均修复时间 | 停机时间 / 故障次数 | 🧮 | 越小越好,反映响应 |
| **停机时长 / 次数** | 当班累计 | ⚙️🧮 | — |
| **停机 Pareto** | 按停机原因排行 | 🧮 | ⭐改善优先级 |
| **报警已持续时长** | now - 报警发生时刻 | 🧮 | ⭐救火时效 |

### 2.5 产线流动类

| 指标 | 公式 / 定义 | 来源 | 说明 |
|------|-----------|------|------|
| **瓶颈站** | 节拍最慢/超 Takt 最多的站 | 🧮 | 限制整线产能 |
| **产线平衡率** | 各站工时和 / (瓶颈站工时 × 站数) | 🧮 | <85% 需调线 |
| **在制品 WIP** | 站间堆积数量 | ⚙️ | 反映流动顺畅度 |
| **断流/堵流** | 上游空/下游满 | 🧮 | 平衡问题信号 |

### 2.6 计划与成本类（主任最关心）

| 指标 | 公式 / 定义 | 来源 | 说明 |
|------|-----------|------|------|
| **计划达成率** | 实产 / 计划 (日/周/月) | 🧮 | ⭐主任C位 |
| **订单/工单进度** | 已完成 / 订单总量 | 🧮📋 | 交付预测 |
| **交付预测** | 按当前速度推算完工时间 | 🧮 | "预计 18:30 完成" |
| **停机损失金额** | 停机损失产能 × 单价 | 🧮📋 | ⭐件数→金额 |
| **质量损失金额** | 不良数 × 单件成本 | 🧮📋 | — |

### 2.7 人员 / 换型 / 物料类（调度关心）

| 指标 | 定义 | 来源 |
|------|------|------|
| **出勤 / 缺勤** | 当班到岗人数 | 📋 |
| **人均产出 (UPPH)** | 产量 / 人数 | 🧮 |
| **换型倒计时** | 换型剩余时间 | 🧮📋 |
| **换型达成 (SMED)** | 实际换型时长 vs 标准 | 🧮 |
| **物料齐套率** | 齐套工单 / 总工单 | 🧮📋 |
| **缺料预警** | 当前库存可支撑时长 | 🧮 | "剩18分钟" |
| **安全运行天数** | 距上次事故天数 | 📋 | 参观屏亮点 |

### 🅰️ 视图一：车间主任屏 —— 经营达成

**使用场景**：办公室大屏 / 手机端，每天扫几次。关心交付、成本、达成趋势。
**核心决策**：今天能否交货？哪条线拖后腿？要不要调资源/追责？
**刷新**：30秒-1分钟即可（非实时救火）

#### 信息分层
- **一眼层**：今日计划达成率红黄绿大灯 + 全车间欠产/落后
- **三秒层**：各产线达成率红黑榜、停机损失金额、综合OEE趋势
- **深看层**：订单进度、7天OEE趋势、质量损失

### 🅱️ 视图二：现场调度/管理员屏 —— 实时救火

**使用场景**：调度中心大屏，盯着看。关心异常、响应时效、协调。
**核心决策**：哪里出事要派人？谁的报警拖太久？哪条线要缺料/换型？

## 第五部分：数据采集反推清单

> 要展示上面的指标，后端/PLC 必须采到这些。给硬件和后端做接口设计参考。

### 设备/PLC 必采（⚙️ 每次过机推送）
```json
{
  "station_id": "S03",
  "timestamp": 1717300000000,
  "pass_count": 1,           // 良品+1
  "fail_count": 0,           // 不良
  "defect_code": null,       // 缺陷代码(开短路/偏移..)← 支撑Pareto
  "cycle_time_ms": 14200,    // 本次节拍 ← 支撑Takt对比
  "equipment_status": 1,     // 0停1运2待3报警
  "alarm_code": null,        // 报警代码 ← 支撑停机Pareto
  "wip_buffer": 5            // 本站缓冲区在制品 ← 支撑WIP
}
```

### 系统配置/基础数据（📋 后台维护）
- 班次时间（含休息、换班）→ 算可用时间、Takt
- 各产线/型号的计划产量、标准节拍、标准UPH
- 单件成本、单价 → 金额化损失
- 缺陷代码字典、停机原因字典
- 物料BOM、库存 → 缺料预警
- 排班/人员名单 → 出勤
- 换型标准时间（SMED基准）

### 后端计算派生（🧮）
- OEE 及 A/P/Q 分解、六大损失
- Pacing（应产、欠产、落后分钟）
- FPY/RTY、Pareto 聚合
- MTBF/MTTR
- 产线平衡率、瓶颈站识别
- 达成率、交付预测

## 第六部分：落地优先级路线图

| 优先级 | 内容 | 价值 | 角色 |
|--------|------|------|------|
| **P0** | PacingCard + AlarmTimer 两个通用组件 | 最高,三角色共用 | 全部 |
| **P1** | 组长屏强化：Takt对比、连续不良、缺陷Pareto、平衡率 | 现有屏升级最快 | 组长 |
| **P2** | 调度屏新建：异常流+计时、缺料、换型倒计时 | 救火刚需 | 调度 |
| **P3** | 主任屏新建：达成灯、红黑榜、金额化损失、趋势 | 经营决策 | 主任 |
| **P4** | 参观屏：大字轮播、俯视图、粒子动效 | 锦上添花 | 参观 |
| **P5** | 视图切换器（4模板）+ 与主题(配色)叠加 | 整合 | 全部 |

> 说明：**模板（指标取舍）** 和 **主题（配色）** 是两个独立维度，可叠加。
> 例：主任视图 + 科技紫主题。
