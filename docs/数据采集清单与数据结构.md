# 数据采集清单与数据结构

> 版本 v1.0 · 配套契约代码：`packages/data-platform/src/contracts/`
> 本文回答一个问题：**整个系统要获取哪些数据、从哪来、什么结构、存哪、给谁用。**
> 看板要展示的指标见《看板信息架构设计》，本文聚焦这些指标背后**必须采到的原始数据**。

---

## 一、数据四大来源

| 标记 | 来源 | 谁负责 | 典型数据 |
|------|------|--------|---------|
| ⚙️ | **设备/PLC 直采** | 边缘网关 | 过机数、节拍、设备状态、缺陷码、报警码、温度/速度等工艺量 |
| 🔗 | **MES 系统** | data-platform（REST 拉取） | 工单、计划数量、交货时间、型号、BOM、工艺路线 |
| 📋 | **基础数据配置** | admin-web 后台 | 工厂/产线/站位档案、班次、目标、阈值、字典、成本单价 |
| 🧮 | **系统计算派生** | data-platform | OEE、Pacing、Pareto、MTBF/MTTR、平衡率、达成率、交付预测 |

> 原则：**能采的采（⚙️）、该问 MES 的问 MES（🔗）、配置的配置（📋）、其余算出来（🧮）**。
> 看板（display-app）只消费 🧮 计算后的结果，不直接碰 ⚙️🔗📋 的原始数据。

---

## 二、完整数据清单

### 2.1 ⚙️ 设备直采（边缘网关采集，每次过机/周期上报）

> 契约：`DeviceTelemetry`（`contracts/device.ts`）。公共字段所有设备都填，`metrics` 按设备类型扩展。

#### 公共字段（所有设备）

| 字段 | 类型 | 说明 | 支撑指标 |
|------|------|------|---------|
| `factoryId/zoneId/lineId/stationId/deviceId` | string | 五级定位 | 路由、聚合 |
| `deviceType` | DeviceType | 设备类型（见下） | metrics 分支 |
| `timestamp` | epoch ms | 采集时刻 | 时序落库 |
| `passCount` | number | 本次良品增量（通常 1） | 产量、UPH、良率 |
| `failCount` | number | 本次不良增量 | 不良率、质量率 Q |
| `cycleTimeMs` | number | 本次节拍 | Takt 对比、表现率 P |
| `equipmentStatus` | 0/1/2/3 | 停机/运行/待机/报警 | 可用率 A、状态灯 |
| `defects[]` | DefectRecord | 缺陷明细（code/坐标） | **缺陷 Pareto** |
| `alarm` | AlarmRecord | 报警/停机码 + 起止 | **停机 Pareto、MTTR、报警计时** |
| `wipBuffer` | number | 本站在制品缓冲 | WIP、断/堵流 |
| `orderNo` / `model` | string | 当前工单/型号（设备若知） | 工单关联 |

#### 设备类型与专属遥测（`metrics`）

| deviceType | 中文 | 专属字段（节选） |
|-----------|------|----------------|
| `solder_printer` | 锡膏印刷机 | 刮刀压力/速度、印刷间隙、锡膏批次 |
| `smt_mounter` | 贴片机 | 贴装点数、CPH、抛料率、异常吸嘴/飞达 |
| `reflow_oven` | 回流焊 | 各温区实测/设定温度、链速、峰值温度、含氧量 |
| `wave_solder` | 波峰焊 | 锡炉温度、预热温度、波峰高度、链速 |
| `aoi` / `spi` | 光学检测 | 检测/通过/判废数、误判率、检测程序 |
| `dip_inserter` | DIP 插件 | 插装点数、漏插率 |
| `ict_fct` | ICT/FCT 测试 | 测试/通过/失败数、失败项、测试程序 |
| `laser_marker` / `generic` | 打标/通用 | 任意键值兜底（`GenericMetrics`） |

> **兼容设计**：新增设备类型 = 加一个 `*Metrics` + 一个联合分支，公共链路零改动。
> 不同协议（Modbus/OPC-UA/MQTT/SECS-GEM）的差异由边缘网关「设备 Profile」吸收，
> 详见《对外接口与通道设计》南向部分。

#### 设备事件（状态变更，独立通道）

> 契约：`DeviceEvent`。遥测是高频流，事件是低频状态变更（上工单、报警、换型），便于 MES/报警订阅。

`status_change` / `alarm_raised` / `alarm_cleared` / `changeover_start|end` / `order_start|end`

### 2.2 🔗 MES 来源（data-platform 拉取 / MES 推送）

> 契约：`MesWorkOrder`（`contracts/mes.ts`）。这是 Pacing、达成率、交付预测、换型的源头。

| 字段 | 说明 | 支撑指标 |
|------|------|---------|
| `orderNo` | 工单号（业务键） | 工单关联 |
| `productModel` / `productCode` | 型号 / 物料编码 | 显示、BOM 关联 |
| `plannedQty` | **计划数量** | 计划产量、Takt、达成率 |
| `dueTs` | **交货时间** | 交付预测、超期预警 |
| `priority` | 优先级 | 看板排序 |
| `targetLineId` | 指定产线 | 工单落线 |
| `standardCycleSec` / `standardUph` | 标准节拍 / UPH | 节拍达成率、表现率 |
| `bomId` | BOM | 缺料预警 |
| `unitCost` | 单件成本 | 金额化损失 |

> 用户明确要的「订单信息、数量、交货时间」= `orderNo` + `plannedQty` + `dueTs`，是入站最小集。

### 2.3 📋 基础数据配置（admin-web 后台维护）

| 类别 | 数据 | 支撑指标 |
|------|------|---------|
| **组织档案** | 工厂/车间/产线/站位/设备树、设备类型与 Profile 绑定 | 全局定位 |
| **班次** | 班次时间表、休息/换班时段、有效工时 | 可用时间、Takt、应产 |
| **目标** | 按班/日/型号的计划产量、标准 UPH、OEE 目标 | 达成率、红黄绿阈值 |
| **阈值** | 节拍超标 %、停机超时、连续不良数、缺料预警分钟 | 报警触发 |
| **字典** | 缺陷代码字典、停机原因字典 | Pareto 归类 |
| **成本** | 单件成本、单价 | 停机/质量损失金额化 |
| **物料** | BOM、库存、消耗速率 | 缺料预警 |
| **人员** | 排班、岗位定员 | 出勤、UPPH |
| **换型** | SMED 标准换型时长 | 换型达成 |
| **轮播** | 大屏视图轮播时长、定屏配置 | display-app 控制 |

### 2.4 🧮 系统计算派生（data-platform 实时计算）

| 指标 | 输入 | 公式 / 逻辑 |
|------|------|------------|
| **OEE / A / P / Q** | 运行时间、产量、节拍、良废 | A×P×Q（见下） |
| **Pacing** | 班次开始、Takt、实产 | 应产=(now−班始)/Takt；欠产=实产−应产；落后分钟=欠产×Takt |
| **缺陷/停机 Pareto** | defects[] / alarm[] | 按 code 聚合排序 |
| **FPY / RTY** | 各站一次通过数 | 站 FPY 连乘 |
| **MTBF / MTTR** | 故障次数、停机时长 | 运行时间÷次数 / 停机÷次数 |
| **产线平衡率 / 瓶颈站** | 各站节拍 | Σ工时 /(瓶颈×站数)；最慢站=瓶颈 |
| **达成率 / 交付预测** | 实产、计划、当前速率 | 实产÷计划；剩余÷速率→完工时刻 |
| **金额化损失** | 损失产能/不良数、单价 | ×成本 |

#### OEE 公式（统一口径）

```
OEE = A × P × Q
A 可用率 = 实际运行时间 / 计划运行时间
P 表现率 = (实产 × 理论节拍) / 实际运行时间
Q 质量率 = 良品数 / 总产出数
```

## 三、数据流转

```
⚙️ 设备 ──协议采集──▶ 边缘网关 ──canonical──▶ EMQX ──▶ data-platform ─┬─▶ TDengine（时序留存）
   (Modbus/OPCUA/...)   (Profile 映射)   (MQTT/TLS)    (计算 OEE/Pacing) ├─▶ Redis（实时快照）
                                                                          ├─▶ WS 推送 ▶ 大屏
🔗 MES ──REST 拉取工单──▶ data-platform ◀──Webhook 回传生产消息── MES   └─▶ PostgreSQL（事件/聚合）
📋 admin-web ──配置──▶ PostgreSQL ──▶ data-platform（计算参数）
```

## 四、数据存储分布

| 存储 | 存什么 | 为什么 | 保留 |
|------|--------|--------|------|
| **TDengine** | ⚙️ 原始遥测时序（每站每次过机） | 亿级写入、按设备分片、降采样 | 1–2 年 |
| **PostgreSQL** | 📋 基础配置 + 🔗 工单 + 🧮 班次聚合/事件/损失 | 结构化、事务、关联查询 | 长期 |
| **Redis** | 🧮 各产线/站位最新状态快照、Pacing 缓存 | 秒级刷新、WebSocket 扇出 | 实时（TTL） |
| **边缘本地（SQLite/文件）** | ⚙️ 断网缓冲 30 分钟 | 离线补传零丢失 | 临时 |

---

## 五、采集频率与时延

| 数据 | 频率 | 端到端时延目标 |
|------|------|--------------|
| ⚙️ 过机遥测 | 每件 / 设备节拍 | 设备→大屏 ≤ 3 秒 |
| ⚙️ 设备事件（报警/换型） | 事件触发即报 | ≤ 1 秒 |
| ⚙️ 网关心跳 | 每 10–30 秒 | — |
| 🔗 MES 工单拉取 | 增量轮询 1–5 分钟 + Webhook 即时 | — |
| 🔗 生产消息回传 MES | 周期 1 分钟 / 完工即时 | — |
| 🧮 Pacing / OEE 计算 | 每次遥测触发增量 | 含在 3 秒内 |
